# Employee-Management-System-Using-SQL
The Employment Management System is a SQL-based database project  designed to manage and organize employee-related information  within an organization.  This system stores and manages: - Employee personal details - Job roles and departments - Salary and bonus structure - Payroll processing - Leave records - Employee qualifications
The database is designed using relational schema with proper 
Primary Keys and Foreign Keys to maintain referential integrity.

This system stores and manages:
- Employee personal details
- Job roles and departments
- Salary and bonus structure
- Payroll processing
- Leave records
- Employee qualifications
  
Key Features:
- One-to-Many relationships between Employee and Payroll, Leave, Qualification
- Use of constraints such as PRIMARY KEY, FOREIGN KEY, UNIQUE, NOT NULL
- Cascading actions to maintain data consistency
- SQL queries for data analysis including salary insights, 
  department analysis, leave tracking, and payroll reporting
- Aggregate functions (SUM, AVG, COUNT, MAX, MIN)
- GROUP BY, HAVING, JOIN operations for reporting

Objective:
To create a centralized, structured, and efficient database system 
that reduces redundancy, improves HR management, and supports 
decision-making through SQL-based analysis.

Outcome:
The system enables accurate payroll calculation, effective leave 
tracking, proper job-role allocation, and optimized salary analysis, 
ultimately improving organizational efficiency.
